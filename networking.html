Homework

  1. Created a command-line program to transfer files from one computer to another computer
    • Create a server and a client and test both with another computer
    • Understand how to use Python's socket module for sending bytes over the network
  2. Establish a UDP Peer to Peer network for pre-existing code of the game Pong in order for two people to play over the network
  3. Create a client to a WordPress XML-RPC server supporting numerous different methods as well as making the client accessible from the command line using Python's argparse module
  4. Establish a gRPC server on a raspberry pi to monitor its own temperature and report values to a client

Projects

  1. Implement two chat systems over the network using the server-client TCP model and the peer-to-peer UDP model
    • Make sure to let each user pick their own username and that it does not match the username of any other user
    • Upon connection, a user receives the past ten messages sent and each message is sent with a time stamp and the user who sent it
    • Make heavy use of Python's socket and argparse modules as well as the asyncio module to be able to run code in parallel
    • Deploy the program onto a raspberry pi inside of a screen session
  2. Implement a store inventory system that can be accessed by a gRPC server/client and a XML-RPC server/client
    • Create all RPC methods yourself and document the API accordingly
    • When the inventory system is shut down, use Python's pickle module to pickle the inventory into a file before termination
    • Create two demonstration clients acting with both servers that share a massive amount of argparse between them
    • Time the client to server interactions in order to see that gRPC runs much faster then XML-RPC because of less network overhead
    • Make sure that the program is screened on an AWS instance
  3. Implement one serial and one distributed program of a genetic algorithm simulation using the traveling salesman problem
    • Make heavy use of Python's celery module to distribute the work of the algorithm to multiple differnet AWS instances
    • Time each trial run to see the difference in efficiency between distributed execution and serial execution
    • Prepare a final presentation of the results including the results of Amdahl's Law and any speedups found
